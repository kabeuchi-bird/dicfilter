# dicfilter

入力テキストファイルから、フィルタファイルに記載された文字列のいずれかを**含む行**を抽出し、出力ファイルへ書き出すツールです。CLI と GUI（egui）の両方に対応します。

## 仕様

- **抽出ルール**: フィルタファイルの各行を 1 つのフィルタ（部分文字列）として扱い、入力ファイルの各行がいずれかのフィルタを含む場合に抽出します。
- **フィルタファイルの書式**: 1 行につき 1 フィルタ。1 行に 2 文字記載があれば、その 2 文字をまとめて 1 つのフィルタとして扱います。空行は無視します。
- **改行方式**:
  - 入力ファイル・フィルタファイル: CRLF / LF 両対応
  - 出力ファイル: 常に CRLF
- **出力ファイル名の衝突回避**: 指定した出力ファイルが既に存在する場合は上書きせず、`output(1).txt`・`output(2).txt` … のように連番を付与した新しいファイルに書き出します。
- **エラー時**: エラーログを標準エラー出力へ、日本語のエラー詳細を出力ファイル内へ記載します。

## CLI

```bash
dicfilter -i 入力.txt -o 出力.txt -f filter.txt
```

| オプション | 説明 | デフォルト |
| --- | --- | --- |
| `-i, --input`  | 入力テキストファイル | （必須） |
| `-o, --output` | 出力ファイル | `output.txt` |
| `-f, --filter` | フィルタファイル | `filter.txt` |

## GUI

引数なしで起動すると GUI が立ち上がります。

```bash
dicfilter
```

- **入力ファイル**: 「選択...」ボタンからダイアログで選択
- **出力ファイル / フィルタファイル**: パスを直接入力（デフォルトはそれぞれ相対パスの `output.txt`・`filter.txt`）
- **実行**ボタンで処理を実行
  - 正常時: 「○○.txtに出力しました」と表示
  - エラー時: 「エラーが発生しました。○○.txtを確認してください」と表示（詳細は出力ファイル内に記載）

> GUI の日本語表示にはシステムにインストールされた CJK フォント（Windows: 游ゴシック/メイリオ/MS ゴシック、macOS: ヒラギノ、Linux: Noto Sans CJK 等）を使用します。

## ダウンロード

各 OS（Linux / Windows / macOS）向けのビルド済みバイナリは、GitHub の [Releases](../../releases) から Zip 形式でダウンロードできます。Zip を展開して実行してください。

## ビルド

ソースからビルドする場合:

```bash
cargo build --release
```

### Linux で GUI をビルドする場合の依存パッケージ

```bash
sudo apt-get install -y libgtk-3-dev libxcb-render0-dev \
  libxcb-shape0-dev libxcb-xfixes0-dev libxkbcommon-dev libssl-dev
```
